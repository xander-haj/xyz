Troubleshooting

Fix the usual setup, build, and missing-asset problems.

Most failures come from missing prerequisites, unsupported ROM input, or package-specific asset paths.

ROM and assets Python packages SDL2 Windows builds Package paths

ROM and assets

Asset extraction needs a compatible USA ROM.

Could not find any ROMs

Place zelda3.sfc or zelda3.smc at the project root, or pass -r with a path relative to the project root.

ROM hash not supported

Normal extraction expects the USA release of A Link to the Past.

Python

Use the project virtual environment for asset tools.

If Python reports No module named PIL or No module named yaml, install requirements.txt into the project venv and run tools through that venv. 

python3 -m venv venv
source venv/bin/activate
python -m pip install -r requirements.txt

SDL2

sdl2-config is required for Make builds.

Install SDL2 development files. On Debian/Ubuntu, use libsdl2-dev. On macOS with Homebrew, install sdl2.

Windows builds

Visual Studio and TCC have different dependency paths.

  • Visual Studio missing SDL2 targets: restore NuGet packages for Zelda3.sln.
  • TCC missing compiler: place tcc.exe at third_party\tcc\tcc.exe.
  • TCC missing SDL: place SDL2.dll at third_party\SDL2-2.26.3\lib\x64\SDL2.dll.
