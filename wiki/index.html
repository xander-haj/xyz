Start here

Pick the path that matches what you want to do.

Z3R does not ship ready-to-play copyrighted game data. Every path eventually needs a compatible user-owned ROM so the project can generate zelda3_assets.dat.

1

Use a release

Download the archive or bundle, then provide the generated asset file it expects.

2

Build from source

Install Git, Python, SDL2, and a C build path, then extract assets and build the native executable.

3

Randomize

Extract clean YAML, run the conservative dungeon chest randomizer, then compile assets again.

Important rule

Bring your own legally dumped USA ROM.

Do not commit, upload, or redistribute zelda3.sfc, zelda3.smc, zelda3_assets.dat, save files, MSU packs, extracted ROM YAML/PNG/audio data, Nintendo artwork, or Nintendo logos. Z3R should distribute the open-source engine and ask users to generate their own runtime assets.

Release builds

What release packages contain.

AssetPlatformNotes
Z3R-windows-x64.zip Windows Contains zelda3.exe, SDL2.dll, zelda3.ini, README, and license.
Z3R-linux-x64.tar.gz Linux Contains the native Linux executable plus zelda3.ini, README, and license.
Z3R-linux-x64.AppImage Linux Uses ~/.local/share/Z3R/zelda3_assets.dat and creates ~/.local/share/Z3R/saves/.
Z3R-macos-intel.tar.gz macOS Intel Native macOS build archive.
Z3R-macos-apple-silicon.tar.gz macOS Apple Silicon Native macOS build archive for arm64 Macs.
Z3R-linux.flatpak Linux Flatpak Installs the app sandbox, but still needs a user-generated asset file.

Release packages are produced by the GitHub Actions workflow when a v* tag is published. They intentionally do not include ROMs or generated assets.

Requirements

Install the right tools for your platform.

Install Apple command line tools, Git, Python 3, Make, and SDL2 development files.

xcode-select --install
brew install git python sdl2
git --version
python3 --version
make --version
sdl2-config --version

On Debian or Ubuntu, install Git, Python venv support, build tools, and SDL2 headers.

sudo apt-get install git python3 python3-venv build-essential libsdl2-dev
git --version
python3 --version
make --version
sdl2-config --version

Install Git for Windows and Python 3 first, then choose Visual Studio, MSYS2, or TCC.

git --version
py --version
python --version

For Visual Studio, install the Desktop development with C++ workload. For TCC, place tcc.exe at third_party\tcc\tcc.exe and SDL2.dll at third_party\SDL2-2.26.3\lib\x64\SDL2.dll.

Source setup

Clone the repo and prepare Python.

git clone --recursive https://github.com/xander-haj/Z3R
cd Z3R

If you already cloned without submodules, initialize them before building.

git submodule update --init --recursive

macOS and Linux venv

python3 -m venv venv
source venv/bin/activate
python -m pip install -r requirements.txt

Windows venv

py -3 -m venv venv
.\venv\Scripts\python -m pip install -r requirements.txt

Asset extraction

Generate zelda3_assets.dat from your ROM.

Place your compatible USA ROM at the project root as zelda3.sfc or zelda3.smc. 

Z3R/
  zelda3.sfc
  assets/
  src/
  zelda3.ini

The normal extraction command writes zelda3_assets.dat to the project root and refreshes extracted files under assets/. 

venv/bin/python assets/restool.py --extract-from-rom
.\venv\Scripts\python assets\restool.py --extract-from-rom

Use the first command on macOS/Linux and the second in Windows PowerShell.

Build and run

Compile the engine after assets exist.

macOS

make -j$(sysctl -n hw.logicalcpu)
./zelda3

Linux

make -j$(nproc)
./zelda3

Windows routes

  • Visual Studio: extract assets, open Zelda3.sln, restore NuGet packages, select Release and x64, then build zelda3.
  • TCC: place TCC and SDL2 in the expected third_party folders, confirm zelda3_assets.dat exists, then run .\run_with_tcc.bat.
  • Moved executables: keep zelda3.ini and zelda3_assets.dat beside the executable unless the package wrapper documents another location.

Randomizer

Current randomizer scope.

The current randomizer is a conservative dungeon chest item shuffle. It does not yet shuffle overworld checks, pots, NPC rewards, shops, boss prizes, entrances, exits, holes, sprites, or room layouts. 

python assets/restool.py --extract-from-rom --no-build
python assets/restool-randomize.py --generate-masterlist
python assets/restool-randomize.py --seed my-first-seed --mode safe
python assets/restool.py
  • The same seed produces the same shuffle.
  • Omitting --seed generates and prints a random seed.
  • Spoiler logs are written to assets/randomizer-spoilers/<seed>.md unless --no-spoiler is used.
  • Small keys and big-key-locked chest locations stay fixed unless you opt in with --include-small-keys or --include-big-chests.
  • Run python assets/restool-randomize.py --restore-vanilla to restore chest YAML from the masterlist.

Configuration

Use zelda3.user.ini for personal settings.

The executable checks zelda3.user.ini first, then falls back to zelda3.ini. 

!include zelda3.ini

[Graphics]
WindowScale = 4
Fullscreen = 0

[General]
ExtendedAspectRatio = 16:9
FillExtendedAspectRatioBorders = 1

[Features]
ItemSwitchLR = 1
NewSettingsMenu = 1

Graphics

Set aspect ratio, renderer, filtering, shaders, sprites, fullscreen, and widescreen HUD behavior.

Sound

Set sample rate, buffer size, MSU mode, MSU path, resume behavior, and volume.

Controls

KeyMap uses SDL key names. GamepadMap uses button names such as A, B, X, Y, L1, and R1.

Features

Enable item L/R switching, settings menu, HUD rearrangement, convenience toggles, and bug-fix modes.

Flatpak users

Install the bundle, then place generated assets in the sandbox data folder.

flatpak install --user ./Z3R-linux.flatpak
flatpak run io.github.xander_haj.Z3R

The current wrapper creates config and save folders, copies the default INI on first run, and exits with a missing-assets message until this file exists: 

~/.var/app/io.github.xander_haj.Z3R/data/Z3R/zelda3_assets.dat

Settings live under ~/.var/app/io.github.xander_haj.Z3R/config/Z3R/, and saves live under the Flatpak data directory.
