Flatpak and Flathub

Install the sandbox cleanly and keep copyrighted assets out of packages.

Z3R Flatpak packages install the engine and wrapper. Users still generate and place their own runtime asset file from a legally dumped compatible ROM.

Install bundle Runtime paths Local package build Flathub submission References

Release bundle

Install the generated Z3R-linux.flatpak file.

flatpak install --user ./Z3R-linux.flatpak
flatpak run io.github.xander_haj.Z3R

The first run may exit with a missing-assets message. That is expected until the user places a generated zelda3_assets.dat in the Flatpak data directory.

Runtime paths

Use Flatpak-safe data and config locations.

PurposePath
Generated assets ~/.var/app/io.github.xander_haj.Z3R/data/Z3R/zelda3_assets.dat
Saves~/.var/app/io.github.xander_haj.Z3R/data/Z3R/saves/
Config~/.var/app/io.github.xander_haj.Z3R/config/Z3R/zelda3.ini

The current wrapper copies the default INI on first run, creates save directories, links config-side asset and save paths back to the data directory, and starts zelda3.real.

Legal boundary

Do not package non-redistributable game data.

Do not place ROMs, zelda3_assets.dat, saves, Nintendo art, Nintendo logos, or generated ROM-derived assets into the Flatpak bundle or a Flathub pull request.

Local build

Build locally on Linux before submitting anywhere.

flatpak run org.flatpak.Builder \
  --force-clean \
  --sandbox \
  --default-branch=stable \
  --user \
  --install \
  --install-deps-from=flathub \
  --ccache \
  --repo=repo \
  builddir \
  packaging/flatpak/io.github.xander_haj.Z3R.yml
flatpak run io.github.xander_haj.Z3R
flatpak run --command=flatpak-builder-lint org.flatpak.Builder manifest io.github.xander_haj.Z3R.yml
flatpak run --command=flatpak-builder-lint org.flatpak.Builder repo repo

Flathub authors

Flathub builds from source.

  • Use app ID io.github.xander_haj.Z3R everywhere package metadata needs an ID.
  • Build make zelda3, not plain make, so CI does not require a ROM.
  • Use file chooser portals for ROM import instead of broad host filesystem access.
  • Add a first-run ROM importer before treating the package as a polished Flathub user experience.
  • Recheck the pinned org.freedesktop.Platform//25.08 runtime before submission.
  • Submit the first app pull request to flathub/flathub:new-pr, not master.
