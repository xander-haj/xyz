Z3R Launcher

Use the desktop launcher to manage local Z3R projects.

The launcher scans, clones, checks, builds, configures, randomizes, and launches Z3R folders. It still requires the user to provide their own legally obtained compatible US ROM.

Overview Use a release Build launcher Project dependencies Windows toolkit

Overview

The launcher is a Tauri 2 desktop app.

Its frontend is plain HTML, CSS, and JavaScript under launcher/src/. Its backend is Rust under launcher/src-tauri/.

Setup

Clone repos, copy uploaded ROMs, create venvs, extract assets, and build projects.

Play

Detect ready builds, launch them, and manage project-specific setup status.

Customize

Edit INI settings, controls, feature toggles, sprites, shaders, MSU packs, and randomizer options.

Recommended path

Use prebuilt launcher installers unless you are developing the launcher.

The launcher README points users to the GitHub Releases tab for prebuilt installers and packages. Those releases do not include a ROM; users must upload their own compatible US .sfc file through the launcher.

Build from source

Build launcher packages on their target operating system.

Shared requirements are Git, Rust with Cargo, Rust 1.77.2 or newer, and Tauri CLI 2.

cargo install tauri-cli --version "^2"
cd launcher
cargo tauri build

Build output is written under launcher/src-tauri/target/release/bundle/.

Z3R project dependencies

Launcher-managed Z3R builds still need platform tools.

  • All platforms need Git, Python 3, a project venv, project Python packages, and a user-owned ROM.
  • macOS needs Xcode Command Line Tools and SDL2 development files.
  • Linux needs Make, a C/C++ toolchain, Python venv support, and SDL2 development files.
  • Windows uses Visual Studio Build Tools for MSBuild, or TCC and SDL2 for the lightweight route.

Windows releases

Windows launcher releases bundle portable setup tools.

The Windows setup exe bundles portable Git, Python, TCC, and SDL2 from launcher/src-tauri/bundled-tools/windows/. MSBuild is still required when a user chooses the Visual Studio build route. 

powershell.exe -ExecutionPolicy Bypass -File ./scripts/prepare-windows-toolkit.ps1
powershell.exe -ExecutionPolicy Bypass -File ./scripts/verify-windows-toolkit.ps1
cargo tauri build
